PHP的require不能乱用，会导致类和函数重复定义错误

09 Oct 2011
  • PHP

今天做了个测试，恩，很简单。写个文件A，在文件A里写两行require文件B的代码，文件B里定义一个class，名字随意。可以发现一个错误：Fatal error: Cannot redeclare class TestForRequire

意思就是说require_once是会检查是否已经引入过这份代码了，但是require则不会，导致require多次的话发生重复定义类的错误。

恩，小经验。
