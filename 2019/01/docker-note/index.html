3. 经验

3.1 简单命令

  • docker images 显示所有镜像
  • docker rmi 删除镜像
  • docker ps -a 可以显示所有的容器，包括不在运行状态的
  • docker rm 删除容器

3.2 tips

  • 做开发的时候可以把镜像构建多包几层，一层层做成镜像存在本地。后面的build可以直接from前面的，节约docker build的时间。否则中间一个步骤出错，很多耗时很长的源码编译工作就要从头来一次，太伤
  • docker里的npm安装记得要加上--unsafe-perm，具体可以看：grpc/grpc-node#604
  • docker run 命令中文化：Docker run 命令参数及使用
  • docker run –rm 会在容器退出时删除这个容器，与 -d 参数冲突
  • docker run -d 会启动一个容器的守护进程，该容器不会退出
  • docker run -i -t $image_name:$version /bin/bash，会启动一个容器，并启动一个终端与该容器连接，可直接操作
